# Deployment Guide — Personal Website

**Primary:** [GitHub Pages](https://pages.github.com/) + custom domain **elotechy.com**  
**Alternative:** Home mini-server + Nginx (see [Appendix: Home Server](#appendix-home-server))

---

## Overview

This is a **static website** — no backend, no database. The main site is pure HTML/CSS/JS. The blog has a lightweight Node.js build step (`blog/build.js`) that converts Markdown → HTML; GitHub Actions runs that build automatically on every push.

```
personal_website/
├── index.html          → https://elotechy.com/
├── site.config.js      → loaded by index.html
├── CNAME               → custom domain for GitHub Pages
├── Eloghosa_CV.pdf     → https://elotechy.com/Eloghosa_CV.pdf
└── blog/
    ├── index.html      → https://elotechy.com/blog/
    ├── posts/*.md      → source files (Markdown)
    ├── html/*.html     → generated by build.js
    └── build.js
```

**Live URLs after deploy:**

| URL | Page |
|-----|------|
| `https://elotechy.com/` | Homepage |
| `https://elotechy.com/blog/` | All published posts |
| `https://elotechy.com/blog/html/<slug>.html` | Individual post |

## GitHub Pages + elotechy.com

### 1. Create the GitHub repo

**`personal_website` is fine** — you do **not** need a repo named `username.github.io`.

| Repo type | Default GitHub URL | With custom domain `elotechy.com` |
|-----------|-------------------|-----------------------------------|
| `personal_website` (project site) | `https://elotech47.github.io/personal_website/` | **`https://elotechy.com/`** (site root — `/blog/` works) |
| `elotech47.github.io` (user site) | `https://elotech47.github.io/` | Same at custom domain |

With **elotechy.com** attached in Pages settings, GitHub serves your project repo at the **domain root**, not under `/personal_website/`. Your existing links (`/`, `/blog/`) work as written.

1. Push this project to your existing **`personal_website`** repo on the `main` branch.

```bash
cd /path/to/personal_website
git init   # skip if already a repo
git add .
git commit -m "Initial site"
git branch -M main
git remote add origin git@github.com:elotech47/personal_website.git
git push -u origin main
```

> **Note:** Before the custom domain is live, the temporary URL is `https://elotech47.github.io/personal_website/`. Absolute paths like `/blog/` only match that URL once `elotechy.com` is configured — use `python3 -m http.server` locally for preview until then.

### 1b. Replacing an old site on elotechy.com

Only **one** GitHub Pages repo can use `elotechy.com` at a time.

1. **Old repo:** **Settings → Pages** → remove **Custom domain** (or delete the old repo).
2. **This repo:** push code → enable **GitHub Actions** as Pages source → set **Custom domain** to `elotechy.com`.
3. DNS can stay as-is if it already points at GitHub’s Pages IPs / CNAME.

### 2. Enable GitHub Pages

In the repo on GitHub:

1. **Settings → Pages**
2. **Build and deployment → Source:** select **GitHub Actions** (not “Deploy from a branch”).
3. After the first push, the **Deploy to GitHub Pages** workflow runs automatically (`.github/workflows/deploy-pages.yml`). It builds the blog, then publishes the whole site.

The site is briefly available at `https://elotech47.github.io/personal_website/` until the custom domain is configured.

### 3. Configure DNS at your registrar

At wherever you manage **elotechy.com** (Cloudflare, Namecheap, Google Domains, etc.), add:

**Apex domain (`elotechy.com`)** — four **A** records:

| Type | Host / Name | Value |
|------|-------------|-------|
| A | `@` | `185.199.108.153` |
| A | `@` | `185.199.109.153` |
| A | `@` | `185.199.110.153` |
| A | `@` | `185.199.111.153` |

**`www` subdomain** — one **CNAME** record:

| Type | Host / Name | Value |
|------|-------------|-------|
| CNAME | `www` | `elotech47.github.io` |

Optional **AAAA** records for IPv6 (GitHub docs list these if you want full IPv6 support).

Verify propagation (may take 15 minutes–48 hours):

```bash
dig elotechy.com +short
dig www.elotechy.com +short
```

### 4. Attach the custom domain in GitHub

1. **Settings → Pages → Custom domain** → enter `elotechy.com` → **Save**.
2. GitHub reads the `CNAME` file in the repo root (already set to `elotechy.com`).
3. Wait for DNS check to pass, then enable **Enforce HTTPS**.

To redirect `www` → apex (or the reverse), set the preferred domain in **Pages → Custom domain**; GitHub handles the redirect once both DNS records exist.

### 5. Commit the CV PDF

`site.config.js` links to `/Eloghosa_CV.pdf`. Add the file to the repo root before pushing:

```bash
git add Eloghosa_CV.pdf
git commit -m "Add CV"
git push
```

### 6. Updating the site

```bash
# Edit site.config.js and/or blog/posts/*.md locally
git add .
git commit -m "Update content"
git push
```

GitHub Actions rebuilds the blog and redeploys. No manual `node build.js` required for production (still useful locally):

```bash
python3 -m http.server 8080   # preview at http://localhost:8080
```

### GitHub Pages checklist

- [ ] Code pushed to **`personal_website`** on the `main` branch
- [ ] **Settings → Pages → Source** is **GitHub Actions**
- [ ] First workflow run succeeded (**Actions** tab)
- [ ] DNS A + CNAME records set for `elotechy.com` / `www`
- [ ] Custom domain `elotechy.com` saved in GitHub Pages settings
- [ ] **Enforce HTTPS** enabled
- [ ] `https://elotechy.com` and `https://elotechy.com/blog/` load correctly
- [ ] `Eloghosa_CV.pdf` committed if the CV link is used

## Appendix: Home Server

The sections below describe self-hosting on a Linux mini-server with Nginx. Use this if you prefer running the site at home instead of (or in addition to) GitHub Pages.

---

## Step 1 — Set Up Git (on your local machine)

Before deploying, version-control the site so you can push updates easily.

```bash
cd /path/to/personal_website

# Initialize a git repo
git init
git add .
git commit -m "Initial site commit"

# Create a .gitignore first
cat > .gitignore << 'EOF'
blog/node_modules/
.DS_Store
*.log
EOF

git add .gitignore
git commit -m "Add gitignore"
```

**Push to GitHub (optional but recommended for backup):**
```bash
# Create a private repo at github.com, then:
git remote add origin git@github.com:yourusername/personal-website.git
git branch -M main
git push -u origin main
```

---

## Step 2 — Prepare the Server

These instructions assume **Ubuntu 22.04 LTS** or **Debian 12** (most common for home servers). Run as a non-root user with `sudo`.

### Install dependencies

```bash
sudo apt update && sudo apt upgrade -y

# Nginx web server
sudo apt install -y nginx

# Certbot (Let's Encrypt SSL)
sudo apt install -y certbot python3-certbot-nginx

# Node.js (for building blog posts from Markdown)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Git (to pull updates)
sudo apt install -y git
```

### Verify installations

```bash
nginx -v       # nginx/1.x.x
node -v        # v20.x.x
certbot --version
```

---

## Step 3 — Deploy the Site

### Clone the repo to your server

```bash
# Create a directory for the site
sudo mkdir -p /var/www/personal_website
sudo chown $USER:$USER /var/www/personal_website

# Clone from GitHub (or copy files manually via SCP)
git clone git@github.com:yourusername/personal-website.git /var/www/personal_website

# OR: if not using GitHub, copy from local machine via SCP:
# scp -r /path/to/personal_website/* user@your-server-ip:/var/www/personal_website/
```

### Build the blog

```bash
cd /var/www/personal_website/blog
npm install          # installs marked + gray-matter (first time only)
node build.js        # generates blog/html/*.html from blog/posts/*.md
```

### Fix permissions

```bash
sudo chown -R www-data:www-data /var/www/personal_website
sudo find /var/www/personal_website -type f -exec chmod 644 {} \;
sudo find /var/www/personal_website -type d -exec chmod 755 {} \;
```

---

## Step 4 — Configure Nginx

Create a new Nginx site configuration:

```bash
sudo nano /etc/nginx/sites-available/personal_website
```

Paste this configuration (replace `yourdomain.com`):

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name yourdomain.com www.yourdomain.com;

    root /var/www/personal_website;
    index index.html;

    # Main site
    location / {
        try_files $uri $uri/ =404;
    }

    # Serve the CV PDF with correct MIME type
    location /Eloghosa_CV.pdf {
        alias /var/www/personal_website/Eloghosa_CV.pdf;
        add_header Content-Type application/pdf;
        add_header Content-Disposition "inline; filename=Eloghosa_CV.pdf";
    }

    # Blog index + posts (blog/index.html lists all published posts)
    location /blog/ {
        try_files $uri $uri/ /blog/index.html =404;
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Cache static assets (fonts, images, CSS, JS)
    location ~* \.(css|js|jpg|jpeg|png|gif|ico|woff|woff2|ttf|svg)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip compression
    gzip on;
    gzip_types text/plain text/html text/css application/javascript application/json;
    gzip_min_length 1024;
}
```

Enable the site:

```bash
# Enable site (symlink to sites-enabled)
sudo ln -s /etc/nginx/sites-available/personal_website /etc/nginx/sites-enabled/

# Remove the default site (optional, keeps things clean)
sudo rm -f /etc/nginx/sites-enabled/default

# Test configuration syntax
sudo nginx -t

# Reload Nginx
sudo systemctl reload nginx
```

**Test:** Visit `http://your-server-ip` — you should see your site.

## Step 5 — Configure Your Domain

### DNS Records (set these at your domain registrar)

After purchasing your domain, log into your registrar (Namecheap, Cloudflare, Google Domains, etc.) and add these records:

| Type  | Host | Value             | TTL  |
|-------|------|-------------------|------|
| A     | @    | `YOUR_SERVER_IP`  | 3600 |
| A     | www  | `YOUR_SERVER_IP`  | 3600 |

> **Find your server's IP:** Run `curl -s ifconfig.me` on the server.
> 
> **Home server note:** Your home IP may be dynamic. If your ISP changes it, you'll need to update the DNS record. Consider using a **Dynamic DNS (DDNS)** service (like DuckDNS or Cloudflare Proxy) for automatic IP updates.

### Home router port forwarding

Your home router must forward external web traffic to your mini-server:

1. Find your server's **local IP**: run `ip addr` on the server (e.g., `192.168.1.50`)
2. Log into your router admin panel (usually `http://192.168.1.1`)
3. Go to **Port Forwarding** and add:
   - Port `80` (HTTP) → `192.168.1.50:80`
   - Port `443` (HTTPS) → `192.168.1.50:443`

DNS propagation takes 15 minutes to 48 hours. Verify with:
```bash
dig yourdomain.com +short        # should return your IP
```

---

## Step 6 — Enable HTTPS (Let's Encrypt SSL)

Once DNS is pointing to your server and port 80 is reachable:

```bash
# Get a certificate (follow the prompts)
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

# Certbot automatically:
# - Obtains the SSL certificate
# - Edits your nginx config to add HTTPS
# - Sets up automatic renewal
```

**Test auto-renewal:**
```bash
sudo certbot renew --dry-run
```

Auto-renewal runs twice a day via a systemd timer. Certificates renew automatically before expiry.

## Step 7 — Updating the Site

### Option A: Manual update (pull + optionally rebuild blog)

```bash
cd /var/www/personal_website
git pull origin main

# If you edited blog posts:
cd blog && node build.js

# Fix permissions after pull
sudo chown -R www-data:www-data /var/www/personal_website
```

### Option B: Simple update script

Create a script for one-command deploys:

```bash
sudo nano /usr/local/bin/deploy-site
```

```bash
#!/bin/bash
set -e
SITE_DIR="/var/www/personal_website"

echo "→ Pulling latest changes..."
cd "$SITE_DIR" && git pull origin main

echo "→ Building blog posts..."
cd "$SITE_DIR/blog" && node build.js

echo "→ Fixing permissions..."
sudo chown -R www-data:www-data "$SITE_DIR"

echo "✓ Deploy complete."
```

```bash
sudo chmod +x /usr/local/bin/deploy-site

# Now deploy with a single command from anywhere:
deploy-site
```

### Option C: Webhook auto-deploy (advanced)

For automatic deploys on every `git push`, use a GitHub webhook + a small listener on your server. Only needed if you push frequently.

## Step 8 — Firewall Configuration

Allow only necessary traffic:

```bash
sudo ufw allow OpenSSH       # Keep SSH access
sudo ufw allow 'Nginx Full'  # Allow HTTP (80) and HTTPS (443)
sudo ufw enable
sudo ufw status
```

## Updating Site Content

All content lives in `site.config.js`. To update:

1. Edit `site.config.js` locally
2. If you edited blog posts (`blog/posts/*.md`), run `node build.js` in `blog/` for local preview
3. Commit and push: `git add . && git commit -m "Update content" && git push`

**GitHub Pages:** Actions rebuilds and redeploys automatically.  
**Home server:** run `deploy-site` (or `git pull` + `node build.js`) on the server.
